1. 概述

标定（Calibration）是机器人操作中的关键步骤，用于确定电机的零位（homing offset）和工作范围（range limits）。LeRobot 中每种机器人和主手设备都实现了自己的标定流程。

核心原则：

  • 标定将电机原始编码器值映射到用户友好的坐标系（角度、-100~100%、0~100%）
  • 标定数据同时存储在电机 EEPROM 和本地 JSON 文件中
  • connect() 时自动检查是否已标定，未标定则自动触发标定流程

2. 标定入口：lerobot_calibrate

文件: src/lerobot/scripts/lerobot_calibrate.py (103 行)

2.1 配置类

@dataclass
class CalibrateConfig:
    teleop: TeleoperatorConfig | None = None  # 主手配置
    robot: RobotConfig | None = None          # 机器人配置

    def __post_init__(self):
        # 互斥检查：必须且只能选择一个设备
        if bool(self.teleop) == bool(self.robot):
            raise ValueError("Choose either a teleop or a robot.")
        self.device = self.robot if self.robot else self.teleop

2.2 标定主流程

@draccus.wrap()
def calibrate(cfg: CalibrateConfig):
    init_logging()

    # 1. 根据配置类型创建对应的设备实例
    if isinstance(cfg.device, RobotConfig):
        device = make_robot_from_config(cfg.device)
    elif isinstance(cfg.device, TeleoperatorConfig):
        device = make_teleoperator_from_config(cfg.device)

    # 2. 连接设备（但不自动标定）
    device.connect(calibrate=False)

    try:
        # 3. 手动触发标定
        device.calibrate()
    finally:
        # 4. 无论成功与否都断开连接
        device.disconnect()

关键点：connect(calibrate=False) 参数防止自动标定，让用户在连接后主动控制标定时机。

2.3 使用示例

# 标定从手机器人
lerobot-calibrate \
    --robot.type=so100_follower \
    --robot.port=/dev/tty.usbmodem58760431541 \
    --robot.id=black

# 标定主手
lerobot-calibrate \
    --teleop.type=so100_leader \
    --teleop.port=/dev/tty.usbmodem58760431551 \
    --teleop.id=blue

3. Robot 基类的标定机制

文件: src/lerobot/robots/robot.py

3.1 connect() 中的自动标定逻辑

class Robot(ABC):
    calibration_dir: Path     # $HF_LEROBOT_HOME/calibration/robots/<name>/
    calibration_fpath: Path   # <id>.json

    def connect(self, calibrate: bool = True):
        self.bus.connect()       # 连接电机总线

        if calibrate and not self.is_calibrated:
            # 如果未标定且 calibrate=True，自动执行标定
            self.calibrate()

    @property
    def is_calibrated(self) -> bool:
        # 检查电机总线层面是否已标定
        return self.bus.is_calibrated

3.2 标定文件管理

# 保存标定数据
def _save_calibration(self, calibration: dict):
    self.calibration_dir.mkdir(parents=True, exist_ok=True)
    with open(self.calibration_fpath, "w") as f:
        json.dump(calibration, f, indent=4)

# 加载标定数据
def _load_calibration(self) -> dict | None:
    if self.calibration_fpath.exists():
        with open(self.calibration_fpath) as f:
            return json.load(f)
    return None

4. SO-100 标定流程详解

以 SO-100 系列（So100Follower / So100Leader）为例，标定流程包含以下步骤：

4.1 SO100Follower 的 calibrate()

# 电机配置 (6 个 STS3215 舵机):
#   shoulder_pan  (ID 1)   - DEGREES 模式
#   shoulder_lift (ID 2)   - DEGREES 模式
#   elbow_flex    (ID 3)   - DEGREES 模式
#   wrist_flex    (ID 4)   - DEGREES 模式
#   wrist_roll    (ID 5)   - DEGREES 模式 (连续旋转)
#   gripper       (ID 6)   - RANGE_0_100 模式

def calibrate(self):
    # 步骤 1: 禁用所有电机扭矩
    self.bus.disable_torque()

    # 步骤 2: 设置位置控制模式
    # 步骤 3: 将手臂移到中间位置，按 Enter
    input("Move the arm to a rest position and press Enter")

    # 步骤 4: 设置半圈归位（Set Half Turn Homings）
    self.bus.set_half_turn_homings()
    # 这会读取当前编码器位置，计算 homing_offset 使得当前位置成为零点

    # 步骤 5: 记录运动范围（Record Ranges of Motion）
    input("Move all joints through their full range of motion, then press Enter")
    self.bus.record_ranges_of_motion()
    # 用户自由移动所有关节，系统记录 min/max 编码器值

    # 步骤 6: 处理 wrist_roll (连续旋转关节)
    # wrist_roll 的范围强制设为 [0, 4095]（全范围旋转）

    # 步骤 7: 写入 EEPROM 并保存 JSON
    calibration = self.bus.read_calibration()
    self._save_calibration(calibration)

BiSOFollower / BiSOLeader 的标定将左右臂分别标定：

5. 电机总线层标定

文件: src/lerobot/motors/motors_bus.py — SerialMotorsBus

5.1 两种标定模式

class SerialMotorsBus:
    def set_half_turn_homings(self):
        """
        将当前位置设为零点
        对每个电机:
          1. 读取当前编码器值 (present_position)
          2. 将 homing_offset 设置为使得当前位置映射到范围中点的值
          3. 写入电机 EEPROM
        """

    def record_ranges_of_motion(self):
        """
        交互式记录运动范围
        对每个电机:
          1. 持续读取编码器值
          2. 记录最小值和最大值
          3. 用户按 Enter 确认
          4. 将 min/max 写入 EEPROM
        """

5.2 归一化模式 (MotorNormMode)

标定数据被用于将原始编码器值映射到用户友好的单位：

class MotorNormMode(Enum):
    RANGE_0_100 = "RANGE_0_100"        # gripper: 原始值 → [0, 100]
    RANGE_M100_100 = "RANGE_M100_100"  # 对称范围: 原始值 → [-100, 100]
    DEGREES = "DEGREES"                # 角度: 原始值 → 度数

# 归一化公式:
# DEGREES: value = (raw - homing_offset) * 360 / 4096
# RANGE_M100_100: value = (raw - min_raw) / (max_raw - min_raw) * 200 - 100
# RANGE_0_100: value = (raw - min_raw) / (max_raw - min_raw) * 100

5.3 标定数据结构

@dataclass
class MotorCalibration:

6. 标定数据持久化

6.1 双层存储

标定数据同时存储在：

  1. 电机 EEPROM（非易失性）:
  • Homing Offset, Min/Max Position Limit
  • 断电后保留，跟随硬件
  1. 本地 JSON 文件:
   {
       "shoulder_pan": {
           "id": 1,
           "drive_mode": 0,
           "homing_offset": 2048,
           "range_min": 0,
           "range_max": 4095
       },
       "shoulder_lift": { ... },
       ...
   }

路径: $HFLEROBOTHOME/calibration/<devicetype>/<devicename>/<id>.json

6.2 首次连接流程

首次连接:
  connect()
    → bus.connect()      # 建立串口通信
    → bus.is_calibrated  # 检查 EEPROM 标定状态
    → False → calibrate() # 自动触发标定
       → set_half_turn_homings()    # 设置零点
       → record_ranges_of_motion()  # 记录范围
       → _save_calibration()        # 保存 JSON

后续连接:
  connect()
    → bus.connect()
    → bus.is_calibrated  # True
    → _load_calibration() # 从 JSON 加载已有的标定参数
    → 不触发 calibrate()
