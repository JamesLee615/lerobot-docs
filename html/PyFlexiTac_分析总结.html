PyFlexiTac 项目分析与功能详解

一、项目概述

PyFlexiTac 是一个用于 FlexiTac 触觉传感器的 Python 接口库。它提供了从 Arduino 固件烧录到传感器数据读取的完整工具链。项目版本为 0.3.1，基于 Python 3.11+，采用 MIT 协议开源。

核心依赖：numpy（数值计算）+ pyserial（串口通信） 可选依赖：matplotlib（实时热力图可视化）

三、四大 CLI 功能详解

3.1 flexitac-find-port —— 查找传感器端口

对应的源码文件：findport.py

它在干什么？

这是一个交互式端口发现工具。当用户不确定传感器连接在哪个 /dev/tty* 设备上时，运行此命令即可找出正确的端口。

工作原理（三步法）：

  1. 快照前：调用 listserialports() 枚举当前系统中所有可用的串口设备（Linux 下扫描 /dev/tty*，Windows 下使用 serial.tools.list_ports.comports()）。
  2. 等待拔插：通过 input() 提示用户拔掉 FlexiTac 的 USB 线缆，然后按回车继续。
  3. 快照后：再次枚举串口，计算 before - after 的差集 —— 消失的那个端口就是传感器所在的端口。

关键实现细节：

def find_port(*, settle_s: float = 0.5) -> str:
    before = list_serial_ports()                          # 拔之前有哪些端口
    input("Unplug the FlexiTac USB cable, then press Enter... ")
    time.sleep(settle_s)                                  # 等 0.5s 让系统稳定
    after = list_serial_ports()                           # 拔之后有哪些端口
    diff = sorted(before - after)                         # 差集 = 消失的端口

异常处理：

情况 行为
恰好 1 个端口消失 返回该端口路径，提示用户重新插上 USB
0 个端口消失 抛出 OSError("No port disappeared...")
多个端口消失 抛出 OSError("Multiple ports disappeared...")

使用方式：

flexitac-find-port
# 或
uv run flexitac-find-port

3.2 flexitac-flash —— 烧录 Arduino 固件

对应的源码文件：flash.py

它在干什么？

这是一个固件编译与烧录工具。它将参数化的 Arduino 固件模板渲染后，通过 arduino-cli 编译并上传到 FlexiTac 传感器连接的 Arduino 开发板上。

工作流程：

模板读取 → 参数替换 → 临时目录写入 → arduino-cli compile → arduino-cli upload

步骤详解：

  1. 模板渲染 (rendertemplate)：读取 template.ino，用正则匹配 #define ROWCOUNT、#define COLUMNCOUNT、#define BAUDRATE、#define MUXCHANNELOFFSET 四行，替换为用户指定的值（或默认值）。
参数 默认值 含义
rows 12 触觉传感行数
cols 32 触觉传感列数
baud 2,000,000 串口波特率
mux_offset 4 多路复用器通道偏移
  1. 板检测 (detect_board)：如果没有通过命令行指定 --port 和 --fqbn，会自动运行 arduino-cli board list 并解析输出，查找唯一连接的 Arduino。恰好一个 → 自动确定；零个 → 报错；多个 → 列出候选让用户选择。
  1. 编译与上传 (flash)：
  • 在临时目录创建 .ino 草图文件
  • 执行 arduino-cli compile --fqbn <fqbn> <sketch_dir>
  • 执行 arduino-cli upload -p <port> --fqbn <fqbn> <sketch_dir>

硬件前提：

  • 需要安装 arduino-cli
  • 需要安装 Arduino AVR 核心：arduino-cli core install arduino:avr
  • Linux 下可能需要串口权限（sudo usermod -aG dialout $USER）

使用方式：

# 自动检测板子和端口
flexitac-flash

# 指定参数
flexitac-flash --port /dev/ttyUSB0 --fqbn arduino:avr:uno

# 非标准传感器尺寸
flexitac-flash --rows 16 --cols 32 --mux-offset 0

3.3 flexitac-stream —— 流式帧数据监控

对应的源码文件：streamframes.py

它在干什么？

这是一个命令行帧数据监控工具。它从传感器逐帧读取数据，并在终端实时打印每帧的关键统计信息——让你在不启动图形界面的情况下，也能快速确认传感器是否正常工作。

它和 heatmap 的区别：

flexitac-stream 是纯文本、轻量级的诊断工具（无需 matplotlib），适合在 SSH 远程连接、无图形界面环境或快速调试时使用。flexitac-heatmap 则是图形化、可视化工具，适合直观地观察压力分布。

工作流程：

打开传感器 → 校准基线 → 逐帧循环 → 打印 FPS / raw_max / norm_max

步骤详解：

  1. 初始化：通过 with FlexiTacSensor(...) as sensor 上下文管理器自动打开串口。
  2. 校准：显式调用 sensor.calibrate()，采集 init_frames（默认 30）帧计算中位数基线。
  3. 循环读取：通过 for i, frame in enumerate(sensor, start=1) 迭代传感器（iter 内部循环调用 read()）。
  4. 打印统计：每帧输出一行，包含四项关键指标。

输出的四项指标含义：

frame=    10 fps=  85.3 raw_max=104 norm_max=0.000
frame=    20 fps=  87.1 raw_max=109 norm_max=0.123
指标 含义 健康范围 异常信号
frame 帧序号（从 1 开始累加） 持续递增 —
fps 自启动以来的平均帧率 ~100+（12×32 传感器 @ 2M baud） 接近 0 或 TimeoutError → 固件未正确发送数据；检查 --rows/--cols/--baud 参数
raw_max 当前帧原始 ADC 最大值（0-255） 静息约 10-40，按压时升高 恒为 0 或 255 → 硬件接线问题
norm_max 当前帧归一化信号最大值（0.0-1.0） 静息 ≈ 0，按压时向 1.0 上升 有按压但不上升 → 基线校准问题或阈值过高

额外功能：按行诊断 (--per-row)

当加 --per-row 参数时：

  • 启动时：打印每行的基线平均值（baseline per row），若某行基线异常偏高/偏低，可能该行接线有问题。
  • 每 20 帧：打印每行的 raw_max，帮助定位"死行"（某一行的所有列读数恒定）。

使用方式：

# 持续运行
flexitac-stream --port /dev/ttyUSB0

# 仅采集 3000 帧后自动退出
flexitac-stream --port /dev/ttyUSB0 --frames 3000

# 按行诊断模式
flexitac-stream --port /dev/ttyUSB0 --per-row

# 非标准传感器尺寸
flexitac-stream --port /dev/ttyUSB0 --rows 16 --cols 32 --baud 115200

命令行参数一览：

参数 默认值 说明
--port (必填) 串口设备路径
--rows 12 触觉行数
--cols 32 触觉列数
--baud 2,000,000 串口波特率
--frames 0 (无限) 采集帧数上限，0 表示持续运行
--per-row False 启用按行诊断输出

3.4 flexitac-heatmap —— 实时热力图可视化

对应的源码文件：visualizeheatmap.py

它在干什么？这是最核心的可视化功能。

flexitac-heatmap 是一个实时触觉压力热力图。它将传感器采集的每一帧数据以伪彩色图像的形式动态渲染在屏幕上，让你可以用眼睛直观地"看到"传感器上的触摸。

更直白的解释：

想象你用手指按压 FlexiTac 传感器表面——屏幕上会实时出现一个彩色矩形，颜色从深蓝（无接触）到亮黄（压力大）变化，亮斑的位置和形状直接反映你在传感器上触摸的位置和力度。这就像一个"热量地图"：哪个区域被按得最重，哪个区域颜色就最亮。

为什么叫 "heatmap"（热力图）？

在数据可视化中，heatmap 是一种用颜色编码数值大小的方法——这里"热"不是指物理温度，而是指压力的"强度"。颜色映射（colormap, 默认 viridis）将归一化后的触觉信号值（0.0~1.0）映射为颜色：

  • 0.0（深色/蓝紫色）→ 无接触，基线水平，传感器处于静止状态
  • 0.5（中间色/绿色）→ 轻微接触
  • 1.0（亮色/黄色）→ 强接触，压力最大

工作流程：

初始化传感器 → 校准基线 → 创建 matplotlib 窗口 → 每 16ms 更新一帧 → 实时渲染

步骤详解：

  1. 初始化与校准：创建 FlexiTacSensor 并显式调用 calibrate()，采集 30 帧计算每个像素的中位数作为基线（baseline）。
  1. 创建 matplotlib 图窗：
  • 一个 AxesImage（imshow）显示触觉阵列的归一化数据
  • 一个 Colorbar 显示颜色与值的映射
  • 一个标题栏显示当前帧序号和实时 FPS
  1. 动画循环 (FuncAnimation, interval=16ms ≈ 60fps)：
  • 每次调用 sensor.read_latest() 获取最新帧（跳过积压帧，保证渲染不滞后）
  • read_latest() 的策略：先读空串口缓冲区的所有待处理数据，取其中最后一帧完整帧；若缓冲区内没有完整帧，则阻塞等待新的完整帧
  • 将 frame.normalized 数组更新到 imshow
  • 计算并更新帧率显示

归一化的含义：

在 sensor.py 的 _normalize 方法中，原始 ADC 读数经过以下处理：

signal = raw - baseline - threshold          # 减去基线和阈值，只保留有效信号
signal = clip(signal, 0.0, 100.0)            # 截断到 [0, 100]
scale = max(signal) + 1e-6                   # 自适应缩放因子（用本帧峰值）
normalized = clip(signal / scale, 0.0, 1.0)  # 归一化到 [0, 1]

这意味着热力图显示的是相对于当前帧最强信号的归一化压力分布——当你按得越重，所有颜色会重新缩放，始终让最大压力处显示为最亮色。这使得热力图对接触的空间分布特别敏感（你在按哪里），而对绝对力度不敏感。

使用方式：

flexitac-heatmap --port /dev/ttyUSB0

# 可选参数
flexitac-heatmap --port /dev/ttyUSB0 --rows 12 --cols 32 --cmap plasma

四、传感器数据协议

FlexiTac 传感器通过串口以 帧协议 持续发送数据：

[0xAA] [0x55] [byte_0] [byte_1] ... [byte_N-1]
  ↑ 帧头标记（2 bytes）     ↑ rows × cols 个 uint8 ADC samples
                           （12×32 = 384 bytes/帧，默认配置）

每个 byte 是 10-bit ADC 读数右移 2 位的结果（即 0-255），代表对应像素的模拟电压值。

六、关键设计决策与亮点

  1. 自适应归一化：使用当前帧的峰值作为归一化缩放因子（而非全局固定值），保证了热力图在不同按压力度下都能清晰显示压力分布。
  1. 帧协议鲁棒性：sensor.py:156-168 的 popframe 方法能处理垃圾字节、分片到达的 marker 等异常情况。
  1. read_latest 机制：sensor.py:128-148 支持跳过积压帧，确保可视化不滞后——这对于实时热力图至关重要。
  1. 参数化固件：模板替换而非代码生成，保持固件代码可读性，同时支持不同尺寸和接线配置的传感器。
  1. 零依赖启动：核心功能仅需 numpy + pyserial，matplotlib 只在需要热力图可视化时作为可选依赖安装。
