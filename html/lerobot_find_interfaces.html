1. 概述

在开始使用 LeRobot 进行机器人操作之前，需要先发现并配置系统中的硬件设备。LeRobot 提供了三个核心脚本来完成这一任务：

脚本 功能 输入 输出
lerobotfindport 发现电机总线的 USB 串口 用户拔插 USB 线 串口路径（如 /dev/ttyUSB0）
lerobotfindcameras 枚举系统中的相机设备 无 相机列表 + 可选的测试截图
lerobotfindjoint_limits 通过遥操作测试关节和末端执行器的工作空间 URDF + 遥控操作 关节角度限制 + EE 位置边界

2. 寻找串口：lerobotfindport

文件: src/lerobot/scripts/lerobotfindport.py (70 行)

2.1 实现原理

采用差分法检测设备：先列出所有可用串口 → 用户拔掉设备 → 再次列出串口 → 对比差异确定目标端口。

def find_port():
    # 第一步：列出所有当前端口
    ports_before = find_available_ports()

    # 第二步：提示用户拔掉 USB 线
    input("Remove the USB cable from your MotorsBus and press Enter when done.")

    # 第三步：再次列出端口
    ports_after = find_available_ports()

    # 第四步：对比差异
    ports_diff = list(set(ports_before) - set(ports_after))

    if len(ports_diff) == 1:
        port = ports_diff[0]  # 唯一差异即为目标端口
    elif len(ports_diff) == 0:
        raise OSError("没有检测到端口差异")
    else:
        raise OSError("检测到多个端口差异")

2.2 端口扫描策略

def find_available_ports():
    if platform.system() == "Windows":
        # Windows: 使用 pyserial 列出 COM 端口
        ports = [port.device for port in list_ports.comports()]
    else:
        # Linux/macOS: 扫描 /dev/tty* 设备
        ports = [str(path) for path in Path("/dev").glob("tty*")]
    return ports

2.3 使用方式

lerobot-find-port

交互流程：

  1. 显示当前所有端口列表
  2. 提示 "Remove the USB cable from your MotorsBus and press Enter when done."
  3. 用户拔掉 USB 线后按 Enter
  4. 等待 0.5 秒确保端口释放
  5. 显示检测到的端口（如 /dev/ttyUSB0）
  6. 提示 "Reconnect the USB cable."

3. 寻找相机：lerobotfindcameras

文件: src/lerobot/scripts/lerobotfindcameras.py (320 行)

3.1 实现原理

支持两种相机类型，使用各自的 find_cameras() 静态方法：

OpenCV 相机发现

def find_all_opencv_cameras():
    opencv_cameras = OpenCVCamera.find_cameras()  # 静态方法
    # Linux: 扫描 /dev/video*
    # 其他: 扫描索引 0-60
    for cam_info in opencv_cameras:
        all_opencv_cameras_info.append(cam_info)

OpenCVCamera.findcameras() 实现（在 src/lerobot/cameras/opencv/cameraopencv.py）：

  • Linux: 遍历 /dev/video* 设备
  • 其他 OS: 遍历索引 0-60
  • 对每个候选设备尝试 cv2.VideoCapture(index) 打开
  • 如果成功打开（cap.isOpened()），记录设备信息并立即释放

RealSense 相机发现

def find_all_realsense_cameras():
    realsense_cameras = RealSenseCamera.find_cameras()

使用 Intel RealSense SDK (pyrealsense2) 的 context.query_devices() 方法查询已连接的 RealSense 设备。

4. 寻找关节限制：lerobotfindjoint_limits

文件: src/lerobot/scripts/lerobotfindjoint_limits.py (222 行)

4.1 目的

通过遥操作控制机器人运动，同时记录其关节角度和末端执行器（EE）位置的工作空间范围。这对于后续配置 IK 安全边界非常有用。

4.2 配置类

@dataclass
class FindJointLimitsConfig:
    teleop: TeleoperatorConfig      # 主手配置
    robot: RobotConfig              # 从手机器人配置
    urdf_path: str                  # URDF 文件路径（用于正运动学）
    target_frame_name: str = "gripper"  # EE 目标帧名称
    teleop_time_s: float = 30       # 录制阶段时长
    warmup_time_s: float = 5        # 预热阶段时长
    control_loop_fps: int = 30      # 控制循环频率

4.3 实现流程

def find_joint_and_ee_bounds(cfg):
    # 1. 连接设备
    teleop = make_teleoperator_from_config(cfg.teleop)
    robot = make_robot_from_config(cfg.robot)
    teleop.connect()
    robot.connect()

    # 2. 初始化正运动学 (placo C++ 库)
    kinematics = RobotKinematics(cfg.urdf_path, cfg.target_frame_name)

    # 3. 预热阶段 (warmup_time_s 秒)
    while elapsed < warmup_time_s:
        action = teleop.get_action()     # 读取主手
        robot.send_action(action)        # 控制从手
        precise_sleep(...)

    # 4. 录制阶段 (teleop_time_s 秒)
    while elapsed < warmup_time_s + teleop_time_s:
        action = teleop.get_action()
        robot.send_action(action)

        # 读取关节角度
        observation = robot.get_observation()
        joint_positions = np.array([obs[f"{key}.pos"] for key in robot.bus.motors])

        # 计算末端执行器位置 (FK)
        ee_pos = kinematics.forward_kinematics(joint_positions)[:3, 3]
        # ee_pos: (x, y, z) 平移分量

        # 更新最小/最大记录
        max_ee = np.maximum(max_ee, ee_pos)
        min_ee = np.minimum(min_ee, ee_pos)
        max_pos = np.maximum(max_pos, joint_positions)
        min_pos = np.minimum(min_pos, joint_positions)

    # 5. 断开设备并输出结果
    robot.disconnect()
    teleop.disconnect()

    print("\n# End Effector Bounds (x, y, z):")
    print(f"max_ee = {r_max_ee}")
    print(f"min_ee = {r_min_ee}")
    print("\n# Joint Position Limits (radians):")
    print(f"max_pos = {r_max_pos}")
    print(f"min_pos = {r_min_pos}")

4.4 关键实现细节

  • RobotKinematics: 使用 placo C++ 库进行 URDF 解析和正运动学计算，位于 src/lerobot/model/kinematics.py
  • 数据流: 主手动作 → 从手执行 → 读取关节角度 → FK 计算 EE 位置 → 更新边界
  • 两阶段设计: 预热阶段让用户确认遥操作正常工作，录制阶段才开始收集数据
  • Ctrl+C 安全: 任何时候按 Ctrl+C 都会安全断开设备并输出已收集的数据
