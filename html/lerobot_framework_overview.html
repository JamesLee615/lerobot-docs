1. LeRobotDataset 的组成

1.1 核心文件结构

LeRobotDataset 在磁盘上的布局如下：

dataset_root/
├── data/                          # 主数据存储 (parquet 格式)
│   ├── chunk-000/
│   │   ├── file-000.parquet       # 每个 parquet 文件包含多帧数据
│   │   ├── file-001.parquet
│   │   └── ...
│   ├── chunk-001/
│   │   └── ...
│   └── ...
├── meta/                          # 元数据
│   ├── episodes/                  # episode 索引信息
│   │   ├── chunk-000/
│   │   │   ├── file-000.parquet
│   │   │   └── ...
│   │   └── ...
│   ├── info.json                  # 数据集信息：特征、形状、fps、视频编码等
│   ├── stats.json                 # 统计信息 (用于归一化)
│   └── tasks.parquet              # 任务描述（语言指令）
└── videos/                        # 视频文件 (可选)
    ├── observation.images.front/
    │   ├── chunk-000/
    │   │   ├── file-000.mp4
    │   │   └── ...
    │   └── ...
    └── observation.images.side/
        └── ...

1.2 LeRobotDataset 类的核心属性

class LeRobotDataset(torch.utils.data.Dataset):
    # 核心属性
    repo_id: str            # HuggingFace Hub 上的数据集 ID
    root: Path              # 本地存储根目录
    meta: LeRobotDatasetMetadata  # 元数据管理器
    hf_dataset: datasets.Dataset  # HuggingFace Dataset 实例 (读取 parquet)
    delta_timestamps: dict  # 时序偏移查询配置
    image_transforms: Callable    # 图像增强变换
    episodes: list[int]     # 加载的 episode 列表 (None = 全部)
    fps: int                # 数据采集帧率
    num_frames: int         # 总帧数
    num_episodes: int       # 总 episode 数
    features: dict          # 特征定义 (键名 → 类型+形状)

1.3 数据特征 (Features) 定义

数据集中的 features 字典定义了每条记录包含的数据字段：

特征键名 FeatureType 含义 示例形状
observation.state STATE 机器人关节角度 (本体感知) [6]
observation.images.* VISUAL 相机图像 [3, 480, 640]
observation.tactile.* STATE (或TACTILE) 触觉传感器数据 [12, 32]
observation.environment_state ENV 环境状态 (仅仿真) 可变
action ACTION 机器人动作 (关节角度) [6]
timestamp 标量 帧的时间戳 [1]
episode_index 标量 所属 episode 索引 [1]
frame_index 标量 episode 内帧索引 [1]
index 标量 全局帧索引 [1]
task 字符串 任务语言描述 标量
next.done 标量 episode 是否结束 [1]
actionispad 布尔 action 是否被填充 (padding) [horizon]

1.4 数据类型和存储方式

  1. Parquet 文件：存储所有非视频数据（状态、动作、时间戳、标量等）。使用 HuggingFace datasets 库加载。
  2. MP4 视频文件：存储图像帧。每个 camera key 对应一个视频文件目录，多个 episode 的视频按顺序存储在同一文件中。
  3. Stats/Info JSON：存储归一化统计量（mean, std, min, max, quantile 等）。
  4. 归一化映射 (normalization_mapping)：
  • VISUAL → MEAN_STD（基于 ImageNet 统计或数据集统计的均值-标准差归一化）
  • STATE → MIN_MAX（最小-最大归一化到 [-1, 1]）
  • ACTION → MIN_MAX
  • TACTILE → MEAN_STD

1.5 Delta Timestamps 机制

deltatimestamps 机制用于从数据集中加载时序相关的帧。每种策略定义自己的 observationdeltaindices 和 actiondelta_indices：

  • Diffusion Policy：
  • observationdeltaindices: [i * framestride for i in range(1 - nobssteps, 1)]（例如 nobs_steps=2 时为 [-3, 0]）
  • actiondeltaindices: [i * framestride for i in range(1 - nobssteps, 1 - nobs_steps + horizon)]
  • ACT Policy：
  • observationdeltaindices: None（仅使用当前帧）
  • actiondeltaindices: list(range(chunksize))（0 到 chunksize-1）

查询时，getqueryindices 方法根据绝对索引和 delta 值计算出需要查询的帧索引，并进行 episode 边界填充（越界帧用边界值填充并标记 ispad）。

1.6 数据加载流程 (getitem)

__getitem__(idx)
    │
    ├── 1. 获取绝对索引和 episode 索引
    ├── 2. _get_query_indices() → 计算需要查询的帧索引
    ├── 3. _query_hf_dataset() → 从 hf_dataset (parquet) 查询状态/动作数据
    ├── 4. _query_videos() → 从 mp4 解码视频帧
    ├── 5. 合并所有数据为字典
    │       { "observation.state": Tensor,
    │         "observation.images.front": Tensor,
    │         "action": Tensor,
    │         "action_is_pad": Tensor,
    │         ... }
    └── 6. 应用 image_transforms (如果提供)

1.7 采样器 (EpisodeAwareSampler)

EpisodeAwareSampler 用于 Diffusion Policy 训练时的特殊采样策略：为了丢弃 episode 末尾因 padding 导致的不完整帧，它会从每个 episode 的有效帧中均匀采样 batchsize 个起始索引，支持 dropnlastframes 参数来丢弃每个 episode 最后的 N 帧。

2. 完整 Pipeline：从标定到评估 (概述)

LeRobot 的完整工作流程分为 6 个阶段，每个阶段有独立的详解文档：

2.1 阶段一：寻找接口 → 📄 lerobotfindinterfaces.md

发现系统中的硬件设备（串口、相机、关节限制）。核心脚本：lerobotfindport（差分法检测 USB 串口）、lerobotfindcameras（OpenCV + RealSense 相机枚举，支持测试拍照）、lerobotfindjoint_limits（通过遥操作 + URDF 正运动学记录关节和 EE 工作空间范围）。

2.2 阶段二：标定 → 📄 lerobotcalibrate.md

对机器人或遥操作设备执行归位和范围标定。流程：connect(calibrate=False) → calibrate()（sethalfturnhomings + recordrangesofmotion）。标定数据双层存储于电机 EEPROM 和本地 JSON。核心类：Robot 基类、SerialMotorsBus、FeetechMotorsBus。

2.3 阶段三：遥操作 → 📄 lerobotteleoperate详解.md

通过主手设备实时控制从手机器人。核心脚本：lerobot_teleoperate。支持 SO-100、Koch、OpenArm、Gamepad、Keyboard、Homunculus 等多种主手类型。处理器管道执行坐标变换、缩放、镜像等操作。

2.4 阶段四：记录数据集 → 📄 lerobotrecord.md

通过遥操作或策略控制机器人并记录所有传感器数据。核心脚本：lerobot_record。支持三种视频编码模式（PNG / Streaming / Batched），数据写入 parquet + mp4。关键类：LeRobotDataset.create()、StreamingVideoEncoder。

2.5 阶段五：训练模型 → 📄 lerobottrain.md

使用离线数据集训练模仿学习策略。核心脚本：lerobottrain。流程：makedataset → makepolicy → makeprepostprocessors → DataLoader → Accelerator.prepare → 训练循环 → 检查点保存。支持多 GPU 分布式训练、RA-BC 加权训练、PEFT/LoRA 微调。

2.6 阶段六：评估模型 → 📄 leroboteval.md

在仿真环境（或真实机器人）中评估预训练策略。核心脚本：leroboteval。关键函数：rollout()（单次推演）→ evalpolicy()（批量评估）→ evalpolicyall()（多环境聚合）。输出 avgsumreward、pc_success 等指标，可选录制评估视频。

3. 数据集处理流程

3.1 数据集工具脚本

LeRobot 提供了多个数据集处理工具：

脚本 功能
lerobotdatasetviz.py 可视化数据集（图像、状态、动作），支持 rerun 在线可视化
lerobotdatasetviz_tactile.py 专门可视化触觉传感器数据
lerobotimgtransformviz.py 预览图像增强效果
leroboteditdataset.py 编辑数据集（删除 episodes、修改任务描述等）
augmentdatasetquantile_stats.py 用分位数统计增强数据集 (用于 TDMPC)
convertdatasetv21tov30.py 从 v2.1 格式转换到 v3.0 格式
compute_stats.py (数据集模块) 计算整个数据集的归一化统计量

3.2 数据集创建流程

1. 初始化 LeRobotDataset (create 模式)
   ├── 创建 root 目录结构
   ├── 初始化 LeRobotDatasetMetadata
   │   ├── info.json (特征定义，fps，总帧数)
   │   ├── stats.json (初始化为空)
   │   └── tasks.parquet
   ├── 创建 data/ chunk 目录
   └── 初始化 TypedWriter (parquet 增量写入器)

2. 对每个 episode 记录:
   ├── add_frame(frame) → 缓冲或直接写入 parquet
   ├── 可选：实时编码视频帧
   ├── save_episode(episode_data)
   │   ├── 更新 meta/episodes 索引
   │   └── 管理 video encoding batch

3. finalize():
   ├── 关闭 parquet writer
   ├── 编码剩余视频 buffer
   ├── 合并同一 camera key 下的多个小 mp4 文件
   └── 计算并保存统计量 (compute_episode_stats)

3.3 数据集加载流程

LeRobotDataset.__init__(repo_id, root, episodes, ...):
│
├── 1. 加载元数据: LeRobotDatasetMetadata
│       └── 从 meta/info.json 读取特征、fps 等
│
├── 2. 加载数据:
│   ├── 本地存在 → load_hf_dataset()
│   │   └── datasets.Dataset 加载 parquet 文件
│   └── 本地不存在 → download("repo_id") → load_hf_dataset()
│
├── 3. 视频下载 (可选):
│   └── 下载 videos/ 目录下的 mp4 文件
│
├── 4. 构建 episode 索引映射:
│   └── _absolute_to_relative_idx: 绝对帧索引 → 相对帧索引
│
└── 5. Delta Timestamps:
    └── 如果配置了 delta_timestamps,
        计算 delta_indices (时序偏移)

3.4 图像增强 Pipeline

图像增强通过 image_transforms 参数传入，支持标准的 torchvision.transforms.v2。训练配置中可定义：

"image_transforms": {
    "enable": true,
    "max_num_transforms": 3,       // 每次最多随机选择 3 个增强
    "random_order": false,
    "tfs": {
        "brightness": {"type": "ColorJitter", "weight": 1.0, "kwargs": {"brightness": [0.8, 1.2]}},
        "contrast":   {"type": "ColorJitter", "weight": 1.0, "kwargs": {"contrast": [0.8, 1.2]}},
        "saturation": {"type": "ColorJitter", "weight": 1.0, "kwargs": {"saturation": [0.5, 1.5]}},
        "hue":        {"type": "ColorJitter", "weight": 1.0, "kwargs": {"hue": [-0.05, 0.05]}},
        "sharpness":  {"type": "SharpnessJitter", "weight": 1.0, "kwargs": {"sharpness": [0.5, 1.5]}},
        "affine":     {"type": "RandomAffine", "weight": 1.0, "kwargs": {"degrees": [-5, 5], "translate": [0.05, 0.05]}}
    }
}

3.5 视频编码流程

记录时:
  PNG 模式:   camera frame → PNG 文件 → (episode 结束后) → encode_video_frames() → mp4
  Streaming:  camera frame → StreamingVideoEncoder (实时 PyAV) → mp4

加载时:
  mp4 → decode_video_frames(timestamps) → torch.Tensor (C, H, W)
  支持的 video_backend: torchcodec (推荐), pyav, video_reader

computeepisodestats() 计算每个 episode 的统计量，然后聚合为全局统计：

stats.json 内容:

optimizerparamgroups.json：

  • 包含每个参数组的优化器配置（lr, betas, eps, weight_decay）和参数索引列表

scheduler_state.json：

  • Cosine 调度器状态：baselrs, lastepoch, stepcount, lastlr

4.4 模型加载流程

# 1. 加载配置
config = PreTrainedConfig.from_pretrained("checkpoints/040000/pretrained_model")
# → 读取 config.json → DiffusionConfig 实例

# 2. 加载策略
policy = DiffusionPolicy.from_pretrained("checkpoints/040000/pretrained_model")
# → 读取 config.json + model.safetensors → DiffusionPolicy 实例

# 3. 加载预处理器
preprocessor = PolicyProcessorPipeline.from_pretrained("checkpoints/040000/pretrained_model")
# → 读取 policy_preprocessor.json + normalizer .safetensors

# 4. 推理
policy.select_action(preprocessor(observation)) → 反归一化动作

4.5 检查点保存机制

checkpoint_dir = get_step_checkpoint_dir(output_dir, total_steps, current_step)
