2. 触觉传感器硬件层

2.1 TactileSensor 类

位于 sensors/tactilesensor.py:43-274

TactileSensor 是对 flexitac.FlexiTacSensor 的封装，通过 USB 串口与触觉传感器通讯。核心功能：

2.1.1 初始化与连接

class TactileSensor:
    def __init__(
        self,
        port: str = "/dev/ttyUSB0",       # USB串口路径
        baud_rate: int = 2_000_000,       # 波特率（固件定义）
        shape: tuple[int, int] = (12, 32),# 触觉阵列 (行, 列)
        baseline: float | None = None,     # 基线值 (None=自动校准)
        init_frames: int = 30,            # 自动校准采集帧数
        threshold: float = 25.0,          # 接触检测阈值 (ADC)
        noise_scale: float = 30.0,        # 低于阈值时的归一化除数
        temporal_alpha: float = 0.2,      # EMA平滑系数 (0-1)
        enable_visualization: bool = True, # cv2实时可视化
    )

连接流程 (connect() → _sensor.open()): 通过 USB 串口打开与 FlexiTac 硬件的连接，设置波特率 2,000,000 bps。

2.1.2 连续读取与线程模型

传感器采用后台线程连续读取模式（continuousread_loop），避免主线程阻塞：

主线程                          后台数据线程                可视化进程
   │                                │                        │
   ├─ start_continuous_read() ──→   │                        │
   │                                ├─ read_latest()         │
   │                                ├─ get_latest_data() ←── │
   │  get_latest_data() ──→ 返回缓存│                        │
   │                                ├─ update_visualization()│
   │                                │   └── mp.Queue ──→    ├─ cv2.imshow()
  • 数据锁 (datalock): 线程安全的 latestdata 访问
  • 停止事件 (stopevent): 优雅终止后台线程
  • 守护线程 (daemon=True): 主进程退出时自动清理

2.1.3 校准系统

校准的目的是建立传感器在无接触状态下的基线值（baseline），所有读数都是相对于基线的差值。

# 自动校准: 采集 init_frames 帧，计算均值作为基线
sensor = TactileSensor(baseline=None, init_frames=30)  # 自动校准

# 固定基线: 跳过校准，所有像素使用相同基线值
sensor = TactileSensor(baseline=20.0)  # 固定基线

# 手动校准
sensor.calibrate(num_samples=30)

# 等待校准完成（阻塞，超时30s）
sensor.wait_for_calibration(timeout_s=30.0)

2.1.4 数据格式

# read_data() 返回归一化到 [0.0, 1.0] 的 numpy 数组
data = sensor.read_data()          # shape: (12, 32), dtype: float32, range: [0, 1]
latest = sensor.get_latest_data()  # 从线程缓存获取最新数据

2.1.5 可视化

可视化在独立子进程中运行（multiprocessing.Process），通过 mp.Queue 接收帧：

# 子进程入口: _visualization_worker()
# 1. 接收归一化的触觉帧
# 2. 应用 cv2.COLORMAP_VIRIDIS 伪彩色映射
# 3. 放大 60× (每个像素 → 60×60 显示块)
# 4. cv2.imshow() 显示

使用了 EMA 时域滤波（temporalfilter）平滑可视化帧，系数由 temporal_alpha 控制。

2.1.6 上下文管理器

with TactileSensor(port="/dev/ttyUSB0") as sensor:
    sensor.start_continuous_read()
    data = sensor.get_latest_data()
# 自动调用 disconnect() 清理资源

4.1.2 get_observation() — 获取观测

覆写了 SOFollower.get_observation()，在原有观测基础上追加触觉数据：

def get_observation(self) -> dict[str, Any]:
    observation = super().get_observation()  # 获得电机位置 + 摄像头图像

    for name, sensor in self._tactile_sensors.items():
        obs_key = f"{OBS_TACTILE}.{name}"       # e.g. "observation.tactile.primary"
        data = sensor.get_latest_data()          # (12, 32) float32 array

        if data is None:  # 容错：传感器无数据时填充零
            data = np.zeros(sensor_cfg.shape, dtype=np.float32)

        observation[obs_key] = data

    return observation

4.1.3 observation_features — 特征声明

声明触觉传感器的观测特征元数据，供数据集和策略使用：

@property
def observation_features(self) -> dict[str, PolicyFeature]:
    features = super().observation_features

    for name, sensor_cfg in self.config.tactile_sensors.items():
        features[f"{OBS_TACTILE}.{name}"] = PolicyFeature(
            type=FeatureType.TACTILE,          # 🔑 标记为 TACTILE 类型
            shape=sensor_cfg.shape,            # (12, 32)
        )

    return features

4.1.4 校准同步

5.2.1 hwtodataset_features() — 硬件特征→数据集特征

这是将机器人硬件特征（类型/形状）转换为 LeRobot 数据集格式的核心函数：

# feature_utils.py:88-154
def hw_to_dataset_features(hw_features, prefix, use_video=True):
    features = {}

    # 1. 分类：关节(flat) vs 摄像头(tuple) vs 触觉(PolicyFeature with TACTILE type)
    joint_fts = {k: v for k, v in hw_features.items()
                 if v is float or (isinstance(v, PolicyFeature) and
                    v.type not in (FeatureType.VISUAL, FeatureType.TACTILE))}
    cam_fts = {k: v for k, v in hw_features.items() if isinstance(v, tuple)}
    tactile_fts = {k: v for k, v in hw_features.items()
                   if isinstance(v, PolicyFeature) and v.type == FeatureType.TACTILE}

    # 2. 处理关节特征 → observation.state / action
    # 3. 处理摄像头 → observation.images.{name}
    # 4. 处理触觉 → observation.tactile.{name}
    for key, pf in tactile_fts.items():
        dim_names = ["height", "width"] if len(pf.shape) == 2 else [...]
        # 避免重复前缀：如果 key 已经是 "observation.tactile.primary" 则不再加前缀
        feature_key = key if key.startswith(f"{prefix}.") else f"{prefix}.tactile.{key}"
        features[feature_key] = {
            "dtype": "float32",
            "shape": pf.shape,      # (12, 32)
            "names": dim_names,     # ["height", "width"]
        }

    return features

关键设计：触觉数据以 float32 类型、2D 形状 (12, 32) 存储在数据集中，用 names: ["height", "width"] 标注维度含义。这与图像（3D shape + video/image dtype）和关节状态（1D shape + float32 dtype）形成对比。

5.2.3 builddatasetframe() — 构建数据帧

# feature_utils.py:157-185
def build_dataset_frame(ds_features, values, prefix):
    frame = {}
    for key, ft in ds_features.items():
        if ft["dtype"] == "float32" and len(ft["shape"]) == 1:
            frame[key] = np.array([values[name] for name in ft["names"]])
        elif ft["dtype"] == "float32":           # ← 触觉走这个分支
            frame[key] = np.array(values[key], dtype=np.float32)  # (12, 32)
        elif ft["dtype"] in ["image", "video"]:
            frame[key] = values[key.removeprefix(f"{prefix}.images.")]
    return frame

触觉数据（2D float32）直接作为 numpy 数组存入帧中，无需额外的图像编解码。

6. 处理器层（Processor Pipeline）

位于 processor/tactileprocessor.py

6.1 TactileValidationProcessorStep

数据格式验证器，确保触觉数据张量的形状正确：

class TactileValidationProcessorStep(ObservationProcessorStep):
    def __init__(self, expected_shape=(12, 32)):
        self.expected_shape = tuple(expected_shape)

    def observation(self, obs):
        for key in obs:
            if not key.startswith(OBS_TACTILE):  # 仅处理触觉键
                continue

            tactile_data = obs[key]
            # numpy → torch
            if isinstance(tactile_data, np.ndarray):
                tactile_data = torch.from_numpy(tactile_data).float()

            # 处理4D: (B, 1, H, W) → (B, H, W) 去除通道维
            if tactile_data.dim() == 4 and tactile_data.shape[1] == 1:
                tactile_data = tactile_data.squeeze(1)

            # 验证最后两个维度 (H, W) 必须匹配期望值
            actual_shape = tuple(tactile_data.shape[-2:])
            if actual_shape != self.expected_shape:
                raise ValueError(f"Expected {self.expected_shape}, got {actual_shape}")

            obs[key] = tactile_data
        return obs

6.2 TactileTemporalFilterProcessorStep

指数移动平均 (EMA) 时域滤波器，用于平滑触觉信号以降低高频噪声：

class TactileTemporalFilterProcessorStep(ObservationProcessorStep):
    def __init__(self, alpha=0.2):
        self.alpha = alpha                    # EMA平滑系数
        self._prev_tactile: dict[str, Tensor] = {}  # 保存每个传感器的上一帧

    def observation(self, obs):
        for key in obs:
            if not key.startswith(OBS_TACTILE):
                continue

            tactile_data = obs[key]  # 转为 torch.Tensor

            # EMA: new = alpha * current + (1 - alpha) * previous
            if key in self._prev_tactile:
                tactile_data = self.alpha * tactile_data + (1 - self.alpha) * self._prev_tactile[key]

            self._prev_tactile[key] = tactile_data.clone().detach()
            obs[key] = tactile_data
        return obs

    def reset(self):
        self._prev_tactile.clear()  # 跨 episode 重置滤波器状态

7. 策略模型层（Policy）

7.1 共享触觉编码器

位于 policies/tactile/encoder.py

这是所有策略模型共用的触觉编码器模块，提供了三种编码器架构。

7.1.1 TactileCNN

基础的 3 层卷积神经网络，将 (B, H, W) 触觉图编码为 (B, feature_dim) 特征向量：

输入: (B, 1, 12, 32)  ← 自动添加通道维
  ↓ Conv2d(1, 32, 3×3) + BN + ReLU + MaxPool2d(2×2)
  ↓ Conv2d(32, 64, 3×3) + BN + ReLU + MaxPool2d(2×2)
  ↓ Conv2d(64, 128, 3×3) + BN + ReLU + MaxPool2d(2×2)
  ↓ Flatten → (B, 128 * H/8 * W/8)
  ↓ Linear → 512 + ReLU + Dropout
  ↓ Linear → feature_dim (256)
输出: (B, feature_dim)

7.1.2 TactileAttentionCNN

带空间注意力机制的 CNN，通过注意力加权突出接触区域：

输入: (B, 1, 12, 32)
  ↓ Conv2d(1, 64, 3×3) + BN + ReLU + MaxPool
  ↓ Conv2d(64, 128, 3×3) + BN + ReLU + MaxPool
  ↓ Conv2d(128, 256, 3×3) + BN + ReLU + MaxPool
  ↓ ┌─ Spatial Attention ──────────────────┐
  ↓ │ Conv2d(256,128,1×1)→ReLU→Conv(128,1)│
  ↓ │               ↓ Sigmoid              │
  ↓ │          attention_weights            │
  ↓ └── × ─────────────────────────────────┘
  ↓ GlobalAvgPool + GlobalMaxPool → concat → (B, 512)
  ↓ Linear → 512 + ReLU + Dropout + Linear → feature_dim
输出: (B, feature_dim)

注意力机制使模型能自动聚焦于触觉图中有接触压力的空间区域。

7.1.3 TactileTokenEncoder

顶层封装，将 CNN backbone 的输出投影为多个 token embedding：

class TactileTokenEncoder(nn.Module):
    def __init__(self, encoder_type, input_shape, feature_dim, n_tokens=1, dropout=0.3):
        # 选择 backbone
        if encoder_type == "cnn":
            self.backbone = TactileCNN(...)
        elif encoder_type == "attention":
            self.backbone = TactileAttentionCNN(...)

        # n_tokens > 1 时，投影到多个 token
        self.token_proj = Linear(feature_dim, n_tokens * feature_dim)

    def forward(self, x: Tensor) -> Tensor:
        # x: (B, H, W)
        feat = self.backbone(x)                       # (B, feature_dim)
        if self.n_tokens == 1:
            return feat.unsqueeze(1)                  # (B, 1, feature_dim)
        feat = self.token_proj(feat)                  # (B, n_tokens*feature_dim)
        return feat.view(B, self.n_tokens, feature_dim)  # (B, n_tokens, feature_dim)

输入输出规范：

  • 输入：(B, H, W) — 触觉帧，自动添加通道维度
  • 输出：(B, ntokens, featuredim) — token 嵌入序列

9.2 触觉渲染管线

# 与 TactileSensor 实时可视化保持一致的渲染管线
def build_tactile_rgb(frame, prev):
    # 1. 自适应归一化
    max_val = np.max(frame)
    if max_val < TACTILE_THRESHOLD (25.0):
        normalized = frame / TACTILE_NOISE_SCALE (30.0)
    else:
        normalized = frame / (max_val + 1e-6)
    normalized = np.clip(normalized, 0.0, 1.0)

    # 2. EMA 时域滤波
    filtered = ALPHA (0.2) * normalized + (1 - ALPHA) * prev

    # 3. uint8 → VIRIDIS 伪彩色 → 放大 60×
    scaled = (filtered * 255).astype(np.uint8)
    bgr = cv2.applyColorMap(scaled, cv2.COLORMAP_VIRIDIS)
    rgb = cv2.cvtColor(bgr, cv2.COLOR_BGR2RGB)
    rgb = cv2.resize(rgb, (w*60, h*60), interpolation=cv2.INTER_NEAREST)

    return rgb, filtered

设计原则总结

  1. 最小侵入：通过继承而非修改原始类来扩展（SOTactileFollower extends SOFollower），保持与原始 LeRobot 框架的兼容性。
  1. 类型系统集成：新增 FeatureType.TACTILE 使触觉成为一等公民，在特征流转的每个环节都有明确标识。
  1. 命名约定：统一使用 observation.tactile.{name} 键名，支持单/多传感器。
  1. 后端复用：TactileTokenEncoder 是所有策略共用的模块，编码器类型（CNN/Attention）通过配置切换。
  1. 容错设计：传感器读取失败时返回零矩阵，确保数据采集不中断。
  1. 元数据完整：校准基线、传感器参数随数据集保存，确保归一化参数可复现。
  1. 传输透明：触觉数据作为观测字典的一部分自然流经 gRPC 传输层，无需特殊处理。
