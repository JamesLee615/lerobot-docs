1. 概述

lerobot_eval 在仿真环境（或真实机器人）中运行预训练策略，执行多个 episode 的 rollout，收集奖励、成功率等指标，并可选地录制评估视频。

两种使用场景：

  1. 训练中评估：lerobottrain.py 在仿真环境中定期调用 evalpolicy_all() 评估检查点
  2. 独立评估：通过 CLI 或脚本直接运行 lerobot-eval 评估已训练的模型

3. eval_main() 主流程

@parser.wrap()
def eval_main(cfg: EvalPipelineConfig):
    # === 1. 初始化 ===
    device = get_safe_torch_device(cfg.policy.device)
    torch.backends.cudnn.benchmark = True
    set_seed(cfg.seed)

    # === 2. 创建环境 ===
    envs = make_env(
        cfg.env,
        n_envs=cfg.eval.batch_size,   # 并行环境数
        use_async_envs=cfg.eval.use_async_envs,
    )
    # envs: dict[suite_name, dict[task_id, VectorEnv]]

    # === 3. 创建策略 ===
    policy = make_policy(
        cfg=cfg.policy,
        env_cfg=cfg.env,
        rename_map=cfg.rename_map,
    )
    policy.eval()  # 评估模式 (关闭 dropout, 使用 running stats)

    # === 4. 创建处理器 ===
    preprocessor, postprocessor = make_pre_post_processors(
        policy_cfg=cfg.policy,
        pretrained_path=cfg.policy.pretrained_path,
        preprocessor_overrides={
            "device_processor": {"device": str(policy.config.device)},
            "rename_observations_processor": {"rename_map": cfg.rename_map},
        },
    )

    # 特定于环境的处理器 (如 LIBERO)
    env_preprocessor, env_postprocessor = make_env_pre_post_processors(
        env_cfg=cfg.env, policy_cfg=cfg.policy
    )

    # === 5. 运行评估 ===
    with torch.no_grad(), torch.autocast(device.type) if cfg.policy.use_amp else nullcontext():
        info = eval_policy_all(
            envs=envs,
            policy=policy,
            env_preprocessor=env_preprocessor,
            env_postprocessor=env_postprocessor,
            preprocessor=preprocessor,
            postprocessor=postprocessor,
            n_episodes=cfg.eval.n_episodes,
            max_episodes_rendered=10,
            videos_dir=Path(cfg.output_dir) / "videos",
            start_seed=cfg.seed,
            max_parallel_tasks=cfg.env.max_parallel_tasks,
        )

    # === 6. 输出结果 ===
    print("Overall Aggregated Metrics:")
    print(info["overall"])   # {avg_sum_reward, pc_success, eval_s, ...}

    # === 7. 保存评估信息 ===
    close_envs(envs)
    with open(cfg.output_dir / "eval_info.json", "w") as f:
        json.dump(info, f, indent=2)

def rollout(
    env: gym.vector.VectorEnv,       # 并行环境 (batch_size 个)
    policy: PreTrainedPolicy,
    env_preprocessor,                # 环境特定预处理
    env_postprocessor,               # 环境特定后处理
    preprocessor,                    # 策略预处理
    postprocessor,                   # 策略后处理
    seeds: list[int] | None = None,
    return_observations: bool = False,
    render_callback: Callable | None = None,
) -> dict:
    """
    Returns:
        "action":   (batch, sequence, action_dim)  动作序列
        "reward":   (batch, sequence)              奖励序列
        "success":  (batch, sequence)              成功标志
        "done":     (batch, sequence)              终止标志 (累计的)
        "observation": (可选) (batch, sequence+1, *) 观察序列
    """

4.2 推演循环

def rollout(env, policy, ...):
    # === 重置环境 ===
    policy.reset()                    # 清空策略的观察/动作缓存
    observation, info = env.reset(seed=seeds)

    all_actions = []
    all_rewards = []
    all_successes = []
    all_dones = []

    step = 0
    done = np.array([False] * env.num_envs)
    max_steps = env.call("_max_episode_steps")[0]

    while not np.all(done) and step < max_steps:
        # === (a) 观察预处理 ===
        observation = preprocess_observation(observation)
        # raw numpy → 转换为 LeRobot 格式 (channel-first tensors)

        # === (b) 添加任务信息 ===
        observation = add_envs_task(env, observation)

        # === (c) 环境特定预处理 ===
        observation = env_preprocessor(observation)

        # === (d) 策略预处理 + 推理 ===
        observation = preprocessor(observation)
        # RenameObs → ToBatch → Device → Normalize

        with torch.inference_mode():
            action = policy.select_action(observation)

        # === (e) 策略后处理 ===
        action = postprocessor(action)  # Unnormalize → CPU

        # === (f) 环境后处理 + 转 numpy ===
        action_transition = {ACTION: action}
        action_transition = env_postprocessor(action_transition)
        action_numpy = action_transition[ACTION].to("cpu").numpy()

        # === (g) 执行动作 ===
        observation, reward, terminated, truncated, info = env.step(action_numpy)

        # === (h) 处理成功标志 ===
        if "final_info" in info:
            successes = info["final_info"]["is_success"].tolist()
        else:
            successes = [False] * env.num_envs

        # === (i) 累积 done 状态 ===
        done = terminated | truncated | done
        if step + 1 == max_steps:
            done = np.ones_like(done, dtype=bool)

        # === (j) 记录 ===
        all_actions.append(torch.from_numpy(action_numpy))
        all_rewards.append(torch.from_numpy(reward))
        all_dones.append(torch.from_numpy(done))
        all_successes.append(torch.tensor(successes))

        step += 1

    # === 堆叠所有时间步 ===
    return {
        ACTION: torch.stack(all_actions, dim=1),    # (B, T, action_dim)
        "reward": torch.stack(all_rewards, dim=1),   # (B, T)
        "success": torch.stack(all_successes, dim=1),# (B, T)
        "done": torch.stack(all_dones, dim=1),        # (B, T)
    }

4.3 关键细节

preprocess_observation() (src/lerobot/envs/utils.py):

def preprocess_observation(observation):
    """将 gym 的原始 numpy 观察转换为 LeRobot 格式"""
    processed = {}
    for key, value in observation.items():
        if "image" in key:
            # 图像: (B, H, W, C) → (B, C, H, W) 且 numpy → torch
            processed[key] = torch.from_numpy(value).permute(0, 3, 1, 2).float() / 255.0
        elif "state" in key:
            processed[key] = torch.from_numpy(value).float()
    return processed

addenvstask(): 从环境的 task 属性中读取任务描述，添加到观察字典中。

done 累积逻辑: 一旦某个环境 done，它保持 done 直到 rollout 结束。这使得后续可以用 mask 过滤掉 done 后的无效数据。

5. eval_policy() 批量评估

文件: src/lerobot/scripts/lerobot_eval.py:251-458

5.1 功能

运行多批次 rollout 以覆盖 n_episodes 个完整 episode：

def eval_policy(env, policy, ..., n_episodes, max_episodes_rendered, videos_dir, ...):
    """
    运行多次 rollout 直到收集到 n_episodes 个 episode 的数据
    """
    # 计算需要的 rollout 批次
    n_batches = ceil(n_episodes / env.num_envs)

    sum_rewards = []
    all_successes = []

    for batch_ix in range(n_batches):
        # 运行一次 rollout
        rollout_data = rollout(env, policy, ...)

        # 找出每个环境第一个 done 的位置
        done_indices = torch.argmax(rollout_data["done"].to(int), dim=1)

        # 创建 mask 过滤 done 后的数据
        mask = (torch.arange(n_steps) <= repeat(done_indices + 1, "b -> b s", s=n_steps)).int()

        # 聚合每个 episode 的 total reward
        batch_sum_rewards = reduce((rollout_data["reward"] * mask), "b n -> b", "sum")
        sum_rewards.extend(batch_sum_rewards.tolist())

        # 聚合每个 episode 的成功状态
        batch_successes = reduce((rollout_data["success"] * mask), "b n -> b", "any")
        all_successes.extend(batch_successes.tolist())

        # 可选: 渲染视频
        if max_episodes_rendered > 0:
            # 异步保存视频 (Thread)
            thread = threading.Thread(target=write_video, args=(video_path, frames, fps))
            thread.start()

    # 聚合最终评估信息
    return {
        "per_episode": [
            {"episode_ix": i, "sum_reward": r, "success": s, ...}
            for i, (r, s) in enumerate(zip(sum_rewards, all_successes))
        ],
        "aggregated": {
            "avg_sum_reward": float(np.nanmean(sum_rewards)),
            "pc_success": float(np.nanmean(all_successes) * 100),
            "eval_s": total_time,
            "eval_ep_s": total_time / n_episodes,
        },
    }

6. evalpolicyall() 多环境聚合评估

文件: src/lerobot/scripts/lerobot_eval.py:681-804

6.1 功能

当存在多个评估环境（如不同的 task suite 或 task id）时，逐个评估并聚合结果。

def eval_policy_all(
    envs: dict[str, dict[int, gym.vector.VectorEnv]],  # {suite: {task_id: vec_env}}
    policy, ...
    n_episodes: int,
    max_parallel_tasks: int = 1,  # 线程池并行数
) -> dict:
    """
    返回:
    {
        "per_task": [{task_group, task_id, metrics}, ...],
        "per_group": {suite_name: {avg_sum_reward, pc_success, ...}},
        "overall": {avg_sum_reward, pc_success, eval_s, ...},
    }
    """
    # 展平任务
    tasks = [(tg, tid, vec) for tg, group in envs.items() for tid, vec in group.items()]

    if max_parallel_tasks <= 1:
        # 顺序评估
        for task_group, task_id, env in tasks:
            metrics = eval_one(env, policy, ...)
            _accumulate(task_group, metrics)
    else:
        # 并行评估 (ThreadPoolExecutor)
        with ThreadPoolExecutor(max_workers=max_parallel_tasks) as executor:
            futures = {executor.submit(run_one, tg, tid, env, ...): (tg, tid)
                       for tg, tid, env in tasks}
            for fut in as_completed(futures):
                tg, tid, metrics = fut.result()
                _accumulate(tg, metrics)

    # 聚合指标
    return {
        "per_task": [...],
        "per_group": {suite: {avg_sum_reward, pc_success, ...}},
        "overall": {avg_sum_reward, pc_success, eval_s, ...},
    }

def _agg_from_list(xs):
    arr = np.array(xs, dtype=float)
    return float(np.nanmean(arr))

overall = {
    "avg_sum_reward": _agg_from_list(all_sum_rewards),
    "pc_success": _agg_from_list(all_successes) * 100,  # 百分比
    "eval_s": total_seconds,
    "eval_ep_s": total_seconds / n_episodes,

7. 仿真环境评估 vs 真实机器人评估

LeRobot 中评估分为两条完全不同的路径，由运行环境决定：仿真用 lerobot-eval，真实机器人用 lerobot-record。

7.1 核心区别

lerobot-eval lerobot-record (策略模式)
运行环境 仿真 (gym) 真实机器人
环境接口 gym.vector.VectorEnv Robot (串口/相机/触觉硬件)
环境创建 make_env(cfg.env) → 自动构建仿真场景 makerobotfrom_config(cfg.robot) → 连接物理设备
动作执行 env.step(action_numpy) → gym 仿真步进 robot.send_action(action) → 电机总线指令
观察获取 env.reset() / env.step() 返回观察 robot.get_observation() → 读电机编码器+相机帧
奖励/成功信号 环境自动返回 reward, is_success 无（真实世界无自动 reward）
并行评估 支持 (gym.vector.VectorEnv 批量并行) 不支持（真实机器人只能串行）
评估指标 avgsumreward, pc_success, 自动统计 无自动指标（需人工观察或后处理）
评估视频 env.render() → 仿真渲染画面 真实相机图像 → logrerundata()
结果产物 eval_info.json + .mp4 视频 LeRobot 数据集（parquet + mp4）
数据集命名约定 不适用 必须 eval 前缀（sanitycheckdatasetname 校验）

7.2 为什么真实机器人不能用 lerobot-eval

lerobot_eval.py 的核心函数 rollout() 硬依赖于 gym 接口：

# lerobot_eval.py — rollout() 中硬编码的 gym 调用
observation, info = env.reset(seed=seeds)              # gym API
observation, reward, terminated, truncated, info = env.step(action_numpy)  # gym API
done = terminated | truncated | done

这些无法替换为真实硬件调用，因为：

  • 没有 Robot 抽象：lerobot-eval 不知道 robot.getobservation() 或 robot.sendaction() 的存在
  • 没有相机管理：不知道如何连接 OpenCV/RealSense 相机
  • 没有触觉传感器：没有 TactileSensor 的读取路径
  • 没有 reward 信号：真实世界需要人工裁定成功/失败，不是 gym 的 is_success 字段

7.4 两种场景的使用方式

场景 A：仿真评估 (lerobot-eval)

# 训练中自动评估（train.py 内部调用）
# 或独立评估：
lerobot-eval \
    --policy.path=lerobot/diffusion_pusht \      # Hub 模型 或本地 checkpoint
    --env.type=pusht \                            # 仿真环境类型
    --eval.batch_size=10 \                        # 并行 10 个环境
    --eval.n_episodes=50 \                        # 总计 50 个 episode
    --policy.device=cuda
# 产物: eval_info.json + videos/*.mp4

场景 B：真实机器人评估 (lerobot-record)

# 纯策略驱动（无遥操作，完全自主）
lerobot-record \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.cameras="{'handeye': {...}, 'fixed': {...}}" \
    --policy.path=outputs/train/act_so101_test/checkpoints/last/pretrained_model \
    --policy.device=cuda \
    --dataset.repo_id=${HF_USER}/eval_so101 \     # eval_ 前缀
    --dataset.single_task="Pick the cube" \
    --dataset.num_episodes=10 \
    --dataset.push_to_hub=false                   # 仅本地评估

# 可选：同时配 teleop 用于安全干预/episode 间重置
# --teleop.type=so101_leader --teleop.port=/dev/ttyACM1

同时配 teleop 和 policy 的场景：

  • 策略是主要动作源（机器人由模型自主控制）
  • 遥操作在 episode 间的重置阶段可用（record_loop() 中 dataset=None 的重置循环）
  • 提供人工安全干预能力
